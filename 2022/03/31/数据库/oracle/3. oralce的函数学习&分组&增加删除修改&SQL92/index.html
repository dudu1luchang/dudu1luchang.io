oracle-多行函数学习

单行函数不改变数据库中的原有数据；

各种函数中有很多，用的时候能想起来有字符函数就好；

有有什么，可以来看看；

多行函数（max,min,avg,sum,count）

对多行数据集中显示，显示出一条；

单行对每一行都改；多行

多行函数的使用
max – 查询员工的最高工资
select max(sal) from emp
min – 查询员工的最低工资
select min(sal) from emp
avg –查询所有的员工工资之和
select avg(sal) from emp;
sum – 查询所有的员工工资之和
select sum(sal) from emp;
count
count(*)-返回表的记录数
count(字段名)-返回非空数
count(distinct 字段名)-去重复后的字段数		 – 查询公司有多少员工
select count(*) from emp
– 查询有津贴员工人数
select count(comm) from emp
【null会自动过滤】
– 查询公司有多少工作种类
select count(distinct job) from emp

sum() avg() 仅适用数值型；

count() max() min() 适用任何类型数据

oracle-分组&分组筛选学习

多行函数使用的时候不能使用单行函数，除非分组；

–查询不同部门最高工资和员工数

– 分组关键字：group by

select 后只能使用group by后的字段【这是规定】

分完组后，之后组叠了；

使用分组是，select后只能使用分组字段和多行函数【这是规定】

当然可以对这个分组字段使用单行函数；

where中不允许使用多行函数；

因为先筛选完成后才能再统计；

where中使用多远函数就会报错；

根据原因是先where后count；

可以使用having来实现；

查询完成后，再筛选再排序；

针对分组后的数据进行筛选；

having必须和分组结合进行使用，不能单独使用；

where 后只能使用单行函数

having后可以使用单行函数也可以使用多行函数；

where的执行效率一定高与having；

所以基础条件的筛选使用having，多行函数的筛选使用having；

where的执行顺序：from->where->group by->select->order by;

having的执行顺序: from->group by ->select->having->order by;

主键：在一张表中，某个字段的值是非空并是唯一的，将这个字段设置为主键；【非空唯一、不重复】

这就是一个约束（保证数据的正确性）

可能表中有多个字段是非空唯一的，选择一个作为主键即可；

主键的作用：唯一的标识一条数据；

前面有几个值，后面必须有几个值；

如果是全字段插入（每个字段都插入），可以省略字段名部分；

如果表中有20个字段，那么就疯了；节省了工作量；

【作为一个过来人，还是建议将字段名写上去】

字段的个数和顺序都要匹配上；

删除数据

delete from 表名 where 条件【删除的是表中的数据】-清空表数据

truncate table dept【也是可以删的】-清空表中数据

truncate table 表名 -删除表中的所有数据，效率高与delete;

drop table 表名 【删除的是表】

部门中有员工的就不让删除；
