学习NIO能够做什么事情？

NIO和IO有相同的作用和目的，但是实现方式不同；

IO是以流的形式处理数据，NIO是以快的形式处理数据（缓冲流）

IO一次一个字节的处理数据，NIO是以快的形式处理数据；

流像水龙头滴水，出来一滴，处理一滴；

块：像一个水壶在接水，接慢后再进行处理

Buffer核心属性_01

这只是简单的存取；

想进一步的掌握buffer缓冲区；

需要掌握四个黑心变量

每个缓冲buffer中都含有四个核心变量（Buffer接口中有四个核心变量）

学习的过程中需要思考，变量都有什么作用；

创建ByteBuffer的时候会指定ByteBuffer的最大容量；

创建ByteBuffer的时候，limit的值就是10；

limit开始后面的位置是不能进行操作的（只能操作前面的位置）；

创建ByteBuffer的时候，下一个要操作的元素就是0这个位置；

注意：capacity一经设定，不能改变；

limit和position的值会随着一些的操作而改变（例如put、get操作）

重点放在前三个属性；

初始化的时候，打印capacity\limit\position这三个值；

经过get\put方法limit及position都是会发生改变的；

limit值是不会发生变化的（因为limit值后的元素是不能进行操作的）；

每次写一个元素，position值是会发生改变的，从零开始；

写六个元素，值就会变成6；

写模式下：下一次进行一个操作，

读模式：要进行一个读取的话：能够读只能读到界线这，读让读几个零食不合适的；读的haul是从position开始读，到limit结束；怎么去读取到javaee这些存进去的数据？（下一节将如何将写模式切断成读模式）

Buffer核心属性_02

capacity已经确定不会改变；

limit与position会根据操作的改变而改变；

虽然put方法没有改变limit的值，调用其他的方法有可能改变的；

要进行读取操作；

读取到的是一片空白，或者几个0；

而我们不是要这些数据，而是存储的数据；

如果将position变成0的话是可以将数据读取出来的；

但是后面还会加上四个000（javaee0000）;

-》所有不仅要改变position的值，还需要改变limit的值；

那如何修改position与limit的值呢；

可以使用flip()方法;

调用这个方法后，limit变成了6，position的值变成了0；

变化的情况：

（1）limit的值会变成position的值；

（2）position变成0；

注意：filp()方法称为切换成读模式；

我们每次要从缓冲区中读取数据时，就要调用filp方法-切换为读模式（否则读的内容会是一片空白）；【重要】

进入到读模式后如何去读写到数据呢？

之前使用的是get(索引的方法)；

这是使用的是get(byte)方法；

需要先创建一个数组；new byte[byteBuffer.limit];(byteBuffer.limit=6 足够了)；

使用byteBuffer.limit的原因，只有byteBuffer.limit多个数据可读；

读取缓冲区中的内容 code
1）创建一个limit()大小的字节数组；(因为只有这个多个数据可读) new byte[byteBuffer.limit()]
2）当读取出来的数据放到字节数组中 byteBuffer.get(bytes)
3）输出数据 new String(bytes,0,bytes.length)

如果想要向缓冲区中再次写入数据;可以使用clear()方法；

get方法执行完成后会影响position的变化；

例如，读模式读取完成后，position的值变成了6；

使用clear方法后，limit的值变成了10，position的值变成了0；

clear():清空缓冲区：核心变量回归写模式，重置核心变量的值；

思考：之前存到缓冲区中的数据，数据是否还存在？

数据时还会存在的，数据处于被遗忘的状态（不能再根据索引的值进行获取了）；

执行clear方法后再次使用get()方法，得到一个数值（74），转化为字符(使用char强转下)为（J）;

这节要清楚读写模式的切换；

Buffer核心属性_03

这里讲解最后一个核心变量：mark;

mark的默认值是-1（标记：一个备忘位置，用于记录上一次的读写位置【position】）
