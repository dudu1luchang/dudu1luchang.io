IO流的概念和分类

io: input and output【输入 and 输入】【读and写】

流：寒流、水流【水流动的状态】、

io流：数据可以像流一样，从一端到另一端的模型；

io的分类：

基础单位不同：

字节流（以字节为单位：1个字节）、字符流（以字符为单位：2个字节）

字节流可以读写任意类型的文件（因为字符流可以拆分为字节流）

字符流只能读写文本文件，里面只能是文字，不能读取图片、视频等；

一般读取有汉字的话建议使用字符流，一次可以读取完整的一个汉字；

按照读写数据的方向不同：

输入流、输出流

输入流-读文件 输出流-写文件

输入流-将数据从文件中读取到内存中；

输出流【从内存中输出】【写文件】-将内存中的数据写入到文件中；

按照流的角色不同：

节点流、处理流

节点流：根文件直接关联的流

处理流：根文件间接关联的流

IO流的框架结构

IO流的框架结构可分为四部分：

字节为单位读，字符为单位读；

字节为单位写，字符为单位写；

转换流：可以将字节流转换成字符流，字符流装换成字节流；

字符编码

给我们的字符指定编号；

因为九三级底层只能识别二进制数据；

a是一个图案，是无法直接在计算机中存储的

给图案a指定一个编数-97；

把字符与整数之间对应关系的表-》编码表；

每个国家的文字都要将其存到计算机中；

常见的编码表：

ASCII：美国标砖信息交换码，使用一个字节的低7位二进制进行表示，因为使用的很少啊；

ISO8859-1：拉丁码表，欧洲码表，使用的文字比美国人多一些，所以使用一个字节的8位二进制表示；

文字多，对应的编号也多，使用的二进制位数也多；

GB2312:中国的中文编码表，汉字比美国人和欧洲人使用的文字要多很多，所以需要编号的范围也比较大，使用的二进制位数也就比较多；最多使用的是两个字节，16位二进制进行表示；

之后对GB2312进行了一个升级-》GBK

GBK:中国的中文编码表（升级版本），融合了更多的中文文字符号；最多使用两个字节16位二进制位表示；

对多：对于英文字母，使用两个字节就太浪费了；

如果出现英文字母还是使用一个字节来表示，如果出现汉字还是两个字节来表示；

那我们如何知道按一个字节来解析，还是两个字节来解析呢？

需要一个标识；

如果二进制位最高位是0：一个字节代表一个编号

如果二进制位最高位是1：两个字节代表一个编号
