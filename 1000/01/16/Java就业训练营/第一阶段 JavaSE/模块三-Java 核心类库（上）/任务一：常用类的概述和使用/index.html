hashCode方法的重写实现

hashCode方法:返回对象的哈希码值；

可以先理解为返回一个内存地址的编号；

可看上面的总结：

用于获取调用对象的哈希码值(内存地址的编号)。若两个对象调用equals方法相等，则各自调用该方法的结果必须相 同若两个调用对象equals方法不相等，则各自调用该方法的结果应该 不相同。为了使得该方法与equals方法保持一致，需要重写该方法。

java官方有规定；约定；

equals判断学号是否相等；

hashCode也依赖于学号就可以了；

耍流氓的方式；不能区分是学号还是hashCode；

这样写就不代表内存地址了；

与学号保持一致性还能与学号进行区分；

包装类的概念和分类 （熟悉）

通常情况下基本数据类型的变量不是对象，为了满足万物皆对象的理念就需要对基本数据类型的变量进行打包封装处理变成对象，而负责将这些变量声明为成员变量进行对象化处理的相关类，叫做包装类。

如：

万物接对象，num违背了万物皆对象的理念；

让变量和类关联起来；

将num打包中到类中；

目的：将基本类型的对象变成对象；

包装类的分类

包装类 对应的基本类型
java.lang.Byte byte
java.lang.Short short
java.lang.Integer int
java.lang.Long long
java.lang.Float ﬂoat
java.lang.Double double
java.lang.Boolean boolean
java.lang.Character char

包装类主要是将我们八种基本类型打包成对象，万物皆对象；
