@EnableAutoConfiguration通过这个注解就可以添加一个属性exclusions,就会将这些给排除出去；

spring.factories（内部有非常多的配置类），不能全部都加载，pom文件中有哪些依赖并且符合注入条件的才进行自动配置；

总结：自动配置步骤（这是为什么springboot能够进行自动配置的原理）

1）启动springboot应用；

2）@SpringBootApplication起作用

3）@EnableAutoConfiguration中有两个注解

4）@AutoConfigurationPackage：中使用@Import(AutoConfigutationPackages,Registrar.class)，通过将Registrar类导入到容器中，而Registrar类的作用是扫描主配置类同级目录及子包，并将相应的组件导入到springBoot创建管理的容器中；

5）@Import(AutoConfigurationImportSelector.class)：通过将AutoCongigurationImportSelector类导入到容器中，AutoConfigurationImportSelector类作用：通过selectImports方法执行的过程中，使用内部工具类SpringFactoriesLoader，查找classpath上所有jar包中的META-INF/spring.factories进行加载，实现将【配置类信息交给SpringFactory加载器进行一系列的容器创建过程；
